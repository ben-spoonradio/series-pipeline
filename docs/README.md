# series-pipeline 문서

이 폴더는 `series-pipeline`의 문서 체계를 관리합니다.

## 문서 운영 워크플로우 (3단계)

1. `docs/worklog/` — 일상 작업 기록 (가볍게, 빠르게)
2. `docs/planning/roadmap.md` — 확정된 계획/릴리스 범위
3. `docs/planning/PRD.md` / `docs/design/` — 복잡한 기능의 상세 스펙/설계

빠른 가이드: 버그/조사/아이디어는 먼저 worklog에 기록하고, 팀이 "이 버전에 넣자"가 확정되면 roadmap으로 승격합니다.

- 워크플로우 상세: `docs/process/workflow.md`
- 개발자 가이드: `docs/implementation/developer-guide.md`
- CLI/옵션 레퍼런스: `docs/implementation/api-reference.md`
